# Tickets (Implementation / Review Log)

This folder stores lightweight tickets used to track implementation and code review items found during project scans.

## Ticket format
Each ticket is a Markdown file named like:

- `TICKET-0001-title.md`

## Ticket workflow folders
Tickets are stored in these folders (ordered):

1. `00_modifications/` — modifications to implement (in repo)
2. `01_in_progress/` — currently being worked on
3. `02_terminate/` — terminated / deprioritized

## Ticket template
Tickets should use the following sections:

- **Area**: (UI / Core / DB / Build / Docs)
- **Functionality**: What capability this ticket refers to
- **Status**: (Open / In Progress / Review Needed / Done)
- **Why**: Problem observed or planned improvement
- **Code references**: file paths + symbols
- **Acceptance criteria**: what “done” looks like
- **Notes**: links, assumptions, follow-ups
